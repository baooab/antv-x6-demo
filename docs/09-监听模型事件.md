# 第 09 课 · 监听 Graph 模型事件

**交互 Demo**：`src/tutorial/demos/Lesson09GraphWatch.tsx` · 页面哈希 `#l09`

## 目标

- 理解 **X6 v3** 上模型的监听方式：**`cell:added`**、**`cell:removed`**，以及 **`cell:change:<属性>`**（例如 **`cell:change:position`**、**`cell:change:data`**），**没有** 名为 **`cell:changed`** 的单一总线事件；
- 在画布旁增加 **实时日志** 与 **`graph.toJSON()`** 预览，观察拖动、改 data、改边端点等操作；
- 对 **`cell:change:position`** 的 **日志** 做节流；**`toJSON` 预览** 仍用短时防抖合并高频变更；
- **React 节点**：业务数据须 **`node.setData(...)`** 写回图模型，才会触发 **`cell:change:data`**；本课输入框写入 **`inputDraft`**。
- **同伴响应（通用）**：使用 **`src/agent-flow/crossNodeDataWatch.ts`** 的 **`bindCrossNodeDataWatch`**——监听 **`cell:change:data`**，在任意节点 data 变化时向 **`getPeers(source, graph)`**（默认：**除来源外全部节点**）合并 **`peerEcho`**（字段名可配）；模块内 **`batch`** 在回写同伴时置位，避免同伴 **`setData`** 再次触发广播形成环路。第 09 课 Demo 含 **A / B / C** 三枚节点，演示非一对一写死。

## 实现要点

- 画布节点为 **`agent-react-card`**（**`createAgentCard`**），带输入框：`AgentReactCard` 内 **`onChange` → `node.setData`**。
- **`bindCrossNodeDataWatch`** 须在 **`graph.on('cell:change:data', …)` 日志监听之前调用并完成注册**，以便同伴回写引起的 **`cell:change:data`** 能通过 **`isPeerBatch()`** 在日志里区分（可选）。
- **`cell:change:attrs`** 仅触发快照刷新、不打文字日志（避免渲染期刷屏）；**`edge:change:source` / `target`** 在连线变更时刷新 **`toJSON`**。

## 下一步（自学）

- 将 **`graph.toJSON()`** 写入状态管理或后端；可按 **`cell:change:data`** 等做增量保存。
- 与官方插件 **History** 配合可实现撤销/重做，同样依赖模型变更。
