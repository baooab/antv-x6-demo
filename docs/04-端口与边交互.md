# 第 04 课 · 端口悬停显隐与边的删除按钮

**交互 Demo**：`src/tutorial/demos/Lesson04PortInteraction.tsx` · 页面哈希 `#l04`

## 目标

- **仅当鼠标悬停在节点上时**显示四向连接桩圆点；**移出节点**后所有桩均隐藏（与是否已连线无关）；
- **悬停时**：已接边的端口为蓝色，未接边为灰色（由是否仍连在边上决定）；
- **边悬停**：显示 **`button-remove`** 工具，便于删除连线；
- 在 **`edge:added` / `edge:removed` / `edge:connected`** 中同步端口颜色与显隐；因移出后全隐藏，需用当前悬停节点 id 判断删边、加边后如何刷新仍被悬停的那枚节点。

## 实现位置

逻辑集中在 **`src/agent-flow/graphInteractions.ts`** 的 **`bindAgentFlowInteractions(graph)`**，与 React 解耦，便于在任意 Demo 或最终 `AgentFlowView` 中复用。

主要辅助函数与状态：

- **`isPortConnected`**：判断某 `portId` 是否仍连在边上；
- **`setPortVisible` / `setPortColor`**：通过 **`node.setPortProp(portId, 'attrs/circle/...', value)`** 改 SVG 属性；
- **`applyNodePortHoverState(graph, node, hover)`**：在「显示/隐藏全部桩」的同时，按连线状态着色；
- 模块内维护 **`hoveredNodeId`**，在 **`edge:added` / `edge:removed` / `edge:connected`** 时若涉及某节点，则按该 id 是否等于 **`hoveredNodeId`** 决定对该节点应用「悬停展开」或「收合隐藏」，避免删边后仍悬停时端口状态错乱；
- 边的 **`addTools` / `removeTools`** 挂在 **`edge:mouseenter`** / **`edge:mouseleave`**（`button-remove`）。
